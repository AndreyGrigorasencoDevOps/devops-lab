# Базовые команды Linux

Справочник основных команд для работы в терминале Linux (Ubuntu, WSL и др.).

---

## Навигация по файловой системе

| Команда | Описание |
|---------|----------|
| `pwd` | Показать текущую директорию (путь) |
| `ls` | Список файлов и папок в текущей директории |
| `ls -la` | Список с деталями (размер, права, дата), включая скрытые файлы |
| `cd путь` | Перейти в указанную директорию |
| `cd ..` | Подняться на уровень выше |
| `cd ~` | Перейти в домашнюю директорию пользователя |
| `cd /` | Перейти в корень файловой системы |

---

## Работа с файлами и папками

| Команда | Описание |
|---------|----------|
| `touch имя_файла` | Создать пустой файл |
| `mkdir имя_папки` | Создать директорию |
| `mkdir -p путь/папка` | Создать директорию и все промежуточные папки |
| `cp источник назначение` | Копировать файл или папку |
| `cp -r папка назначение` | Копировать папку рекурсивно (со всем содержимым) |
| `mv источник назначение` | Переместить или переименовать файл/папку |
| `rm файл` | Удалить файл |
| `rm -r папка` | Удалить папку и всё содержимое |
| `rm -rf папка` | Удалить папку принудительно, без подтверждения (осторожно!) |
| `cat файл` | Вывести содержимое файла в терминал |
| `head -n 10 файл` | Показать первые 10 строк файла |
| `tail -n 10 файл` | Показать последние 10 строк файла |
| `less файл` | Просмотр файла постранично (выход: `q`) |

---

## Поиск и права

| Команда | Описание |
|---------|----------|
| `find . -name "*.js"` | Найти файлы по имени (например, все .js в текущей папке и ниже) |
| `grep "текст" файл` | Найти строки, содержащие «текст», в файле |
| `grep -r "текст" .` | Искать «текст» во всех файлах в текущей папке рекурсивно |
| `chmod +x файл` | Сделать файл исполняемым |
| `chmod 755 файл` | Установить права: владелец — всё, остальные — чтение и выполнение |
| `chown пользователь:группа файл` | Изменить владельца и группу файла |

## Система и процессы

| Команда | Описание |
|---------|----------|
| `whoami` | Имя текущего пользователя |
| `uname -a` | Информация о системе (ядро, архитектура) |
| `df -h` | Свободное место на дисках (в человекочитаемом виде) |
| `du -sh папка` | Размер папки |
| `ps aux` | Список запущенных процессов |
| `top` | Интерактивный монитор процессов (выход: `q`) |
| `kill PID` | Завершить процесс по ID |
| `kill -9 PID` | Принудительно завершить процесс |

---

## Сеть и загрузки

| Команда | Описание |
|---------|----------|
| `curl http://адрес` | Сделать HTTP-запрос, вывести ответ в терминал |
| `curl -O URL` | Скачать файл по URL в текущую папку |
| `wget URL` | Скачать файл по URL |
| `ping хост` | Проверить доступность хоста (остановка: `Ctrl+C`) |
| `ip a` или `ifconfig` | Сетевые интерфейсы и IP-адреса |

---

## Переменные и окружение

| Команда | Описание |
|---------|----------|
| `echo $VAR` | Вывести значение переменной окружения |
| `export VAR=значение` | Установить переменную для текущей сессии |
| `env` | Показать все переменные окружения |
| `which команда` | Путь к исполняемому файлу команды (например, `which node`) |

---

## Полезные сочетания

| Действие | Описание |
|----------|----------|
| `Ctrl+C` | Прервать выполнение команды |
| `Ctrl+L` | Очистить экран терминала |
| `Ctrl+Z` | Приостановить процесс (продолжить: `fg`) |
| `↑` / `↓` | Листать историю команд |
| `Tab` | Автодополнение пути или команды |
| `команда > файл` | Перенаправить вывод команды в файл (перезапись) |
| `команда >> файл` | Дописать вывод в конец файла |
| `команда1 | команда2` | Передать вывод первой команды на вход второй (пайп) |

## Для этого проекта (Node.js в WSL)

```bash
# Перейти в папку проекта (путь в WSL обычно /mnt/c/Users/...)
cd /mnt/c/Users/marty/Desktop/cloud-task-manager/node-api

# Установить зависимости
npm install

# Запустить приложение
npm start

# Собрать Docker-образ
docker build -t node-api .

# Запустить контейнер
docker run -d -p 3000:3000 --name node-api-app node-api
```
